Core Functional Flows

1

Doctor Inbox Management

Login as Doctor/Admin → Campaigns → Pick campaign → Doctor Inbox tab

Search & Filter
Type a child's name in the search bar. Use Risk Level chip filters (No Risk, Risk Detected). Verify results update in real-time.
Module Grouping
Expand a child row. Verify observations are grouped under VITALS and HEAD-TO-TOE EXAMINATION section headers.
Bulk Approve
Click the green "Approve Normal (N)" button in the header. Verify it auto-approves all no-risk observations. Count should update.
Prev/Next Navigation
Expand a child. Use the ← Prev and Next → arrow buttons. Verify it cycles through children in the list.
Triage Decision
Click Approve, Refer, or Follow-up. Add notes. Verify the decision saves and badge updates.
Report Link
Click the "Report" link in the expanded toolbar. Verify it opens the Child Report page.
2

Child Health Report

From Doctor Inbox → click Report link, or navigate directly

Visual Components
  • ✓ Gradient Header: Blue gradient with child name, age, gender, class, campaign
  • ✓ Risk Badge: Green (All Clear), Amber (Review Suggested), Red (Needs Attention)
  • ✓ Stats Strip: Modules Screened, Normal, Findings, 4D Categories
  • ✓ Growth Chart: Toggle between Height, Weight, BMI tabs
  • ✓ Vitals Table: Module, Value, Status, Review columns
Condition Cards (5 sections each)
  • 1. What to Look For — symptoms description
  • 2. How Common Is This — sky-blue card with prevalence
  • 3. Prevention Tips — actionable prevention
  • 4. What You Can Do — amber card with care steps
  • 5. When to Talk to a Doctor — rose card with warning signs
4D Clinical Summary

Should only appear if conditions are detected (status = "present"). Categories: Defects, Delay, Disability, Deficiency, Behavioral, Immunization, Learning. Each shows condition name, ICD code, severity badge, and description. If child is "All Clear", this entire section should be hidden.

Print Formatting: Click "Print Report" button → verify clean A4 layout with page breaks
