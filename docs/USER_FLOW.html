User & Information Flow

Who acts on the SKIDS Screen V3 system, what they produce, what they see, and how a single child's data moves from first screening to research aggregate — with consent gates and DPDP-aligned parent rights throughout.

Who uses the system

Six distinct roles. Each has a separate surface, separate write permissions, and a different view of the same child.

📱 Nurse / Field Screener

Expo mobile app · PIN login
  • Registers child (name, DOB, gender, class, parent phone)
  • Captures observations — photo, video, audio, values, forms
  • Adds annotation chips + quality notes
  • Sees own campaigns + children assigned
  • Sees Tier 1/2 on-device AI suggestions
  • Triggers device readiness checks before a campaign

👩‍⚕️ Doctor / Reviewer

Web dashboard · Mobile review
  • Doctor inbox — observations queued for review
  • Tier 1 + Sandbox second-opinion annotations side-by-side
  • Evidence panel + similar past cases (Phase 7)
  • Writes clinician_review JSON (confirm / refute / add dx)
  • "Request second opinion" button per observation
  • Signs off → triggers report release

👨‍👩‍👧 Parent

Web portal via QR/WhatsApp link
  • Views their child's report (HTML now, PDF after Phase 3)
  • Gives / revokes consent per study
  • Can request data export (DPDP right)
  • Can request data deletion (DPDP right)
  • Receives BHASH WhatsApp notifications

🛠 Admin / Org Owner

Web admin pages
  • Creates campaigns, invites screeners, assigns doctors
  • Manages AI config (failover chain, feature flags)
  • Releases reports (toggles campaigns.reports_released)
  • Sees audit log, AI usage, device status
  • Archives campaigns to R2

🏛 Authority / District

Authority dashboard
  • Aggregate population health by campaign / age / gender
  • Red-flag prevalence, cohort trends (Phase 4 DuckDB)
  • Screener throughput, SLA compliance
  • No row-level PHI — aggregates only

🔬 Researcher (IRB-approved)

DuckDB local or MotherDuck (deferred)
  • Queries de-identified, consent-gated study slices
  • Joins observations, annotations, accuracy_metrics
  • Per-study pseudonymous child IDs (HMAC-salted)
  • Cannot write back to production

Parent rights — DPDP Act obligations in data flow
